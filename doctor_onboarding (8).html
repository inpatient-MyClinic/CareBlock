🗄 Database Settings

Connect a real database so all submissions are stored permanently.

One-time Railway setup:
1. In Railway → New Project → Deploy from GitHub repo using the supplied server.js + package.json (or upload them).
2. In the same project → + New → Database → Postgres. Railway auto-injects DATABASE_URL.
3. Under the service → Settings → Networking → Generate Domain. Copy the URL (e.g. https://your-app.up.railway.app) into the field above.
4. Click 🔌 Test connection → 💾 Save & Activate.
The doctors table is created automatically on first start.
One-time Supabase setup:
1. Create a free project at supabase.com.
2. In SQL Editor, run once:
create table doctors (id text primary key, created_at bigint, data jsonb); alter table doctors enable row level security; create policy "allow all" on doctors for all using (true) with check (true); 3. Settings → API → copy the Project URL and anon public key above.
